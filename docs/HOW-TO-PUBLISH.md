# Как опубликовать

## Публикация нового гайда

1. Создайте гайд по инструкции в [HOW-TO-ADD-GUIDE.md](HOW-TO-ADD-GUIDE.md)
2. Добавьте файлы в git:
   ```bash
   git add docs/guides/ваш-slug/
   ```
3. Сделайте коммит:
   ```bash
   git commit -m "feat: add guide — название гайда"
   ```
4. Запушьте в main:
   ```bash
   git push
   ```
5. GitHub Pages автоматически задеплоит через 1–2 минуты

## Проверка после публикации

Ваш гайд будет доступен по адресу:

```
https://image774477.github.io/Guides/guides/ваш-slug/
```

**Проверьте:**
- Страница открывается без 404
- CSS и JS загрузились (тёмная тема видна, чекбоксы кликаются)
- OG-превью: отправьте ссылку в Telegram — должны показаться заголовок, описание и обложка

## Проверка OG-превью

Отправьте ссылку на гайд в Telegram-чат (можно себе в «Избранное»). Telegram покажет карточку с:
- Заголовком (og:title)
- Описанием (og:description)
- Обложкой (og:image) — если есть `cover.webp` в папке гайда

Если превью не появляется:
- Подождите 2–3 минуты после деплоя
- Убедитесь что canonical URL и OG-теги указывают на правильный адрес
- Попробуйте отправить ссылку ещё раз (Telegram кэширует превью)

## Обновление существующего гайда

1. Отредактируйте `index.html` в папке гайда
2. `git add`, `git commit`, `git push`
3. GitHub Pages обновит автоматически

## Troubleshooting

**Страница показывает 404:**
- Проверьте, что папка гайда в `docs/guides/` (не в корне)
- Проверьте, что файл называется `index.html` (не `Index.html`)
- Убедитесь, что GitHub Pages включён: Settings → Pages → Source: main / /docs

**CSS/JS не загружаются:**
- Проверьте пути: должны быть `../../assets/css/guide.css` (два уровня вверх)
- Убедитесь что `.nojekyll` есть в `docs/`

**Страница деплоится, но выглядит сломано:**
- Откройте DevTools → Console и Network, найдите 404 ошибки загрузки файлов
- Сравните пути с рабочим гайдом `hades-2-platinum`
